# CF-1B 坐标转换

今天分析一道 Codeforces 的坐标转换题。题目要求把电子表格的两种坐标表示法互相转换。

## 两种坐标格式

第一种是 Excel 风格：`BC23` 表示第 55 列第 23 行。列用字母（A-Z, AA-AZ...），行用数字。

第二种是 `R23C55` 格式，直接标明行号和列号。

## 如何判断输入格式

观察样例：
- `R23C55` → 以 R 开头，后面数字，然后 C，再后面数字
- `BC23` → 以字母开头，后面数字

判断方法：如果字符串符合 `R数字C数字` 模式，就是第二种格式；否则是第一种。

## 转换逻辑

**第一种 → 第二种（BC23 → R23C55）**

需要把字母部分转成数字。BC = 2×26 + 3 = 55？验证：A=1, B=2, C=3，所以 BC = 2×26 + 3 = 55。对。

核心算法：遍历字母，`res = res × 26 + (ch - 'A' + 1)`

**第二种 → 第一种（R23C55 → BC23）**

需要把数字 55 转成字母。这是上面过程的逆运算。

55 ÷ 26 = 2 余 3 → 最后一位是 C
2 ÷ 26 = 0 余 2 → 第一位是 B

所以是 BC。

注意：这不是普通的进制转换！因为 A 对应 1 而不是 0。

比如 26：如果直接除，26 ÷ 26 = 1 余 0，余 0 没法对应字母。所以转换前先减 1：

```
while (n > 0) {
    n--;           // 关键：先减 1
    ans += 'A' + (n % 26);
    n /= 26;
}
```

这样 26→25→Z，然后 n=0，结束。52 也能正确得到 AZ。

## 代码核心

```cpp
// 数字转列名（55 → "BC"）
string num2col(long long n) {
    string res;
    while (n > 0) {
        n--;
        res += 'A' + (n % 26);
        n /= 26;
    }
    reverse(res.begin(), res.end());
    return res;
}

// 列名转数字（"BC" → 55）
long long col2num(const string& s) {
    long long res = 0;
    for (char c : s) {
        res = res * 26 + (c - 'A' + 1);
    }
    return res;
}
```

主函数里根据格式判断，调用相应的转换即可。

## 总结

核心要点：
1. 26进制但不是标准进制，A=1 而不是 A=0
2. 数字转字母时要先减 1 再取模，这是最容易错的地方
3. 用正则或简单判断区分两种格式

时间复杂度 O(n × L)，L 是字符串长度。
