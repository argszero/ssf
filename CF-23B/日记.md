# CF-23B 派对

今天分析一道 Codeforces 的思维题。题目说 $n$ 个人参加聚会，没有朋友的人先离开，然后是有 1 个朋友的人离开，接着是有 2 个朋友的人离开... 求最后最多能剩下多少人。

## 理解题意

关键在于"朋友关系"是我们可以自由构造的。目标是让尽可能多的人留下来，也就是让尽可能多的人在某一轮之前都不离开。

一个人有 $k$ 个朋友，他会在第 $k+1$ 轮离开（第 0 轮是没有朋友的人离开）。

## 分析样例

$n=3$，答案是 1。

如果 3 个人互相都是朋友，每个人有 2 个朋友。第 0 轮：没人离开（都有朋友）。第 1 轮：没人离开（都有 2 个朋友，不是 1 个）。第 2 轮：3 个人都有 2 个朋友，全部离开。剩下 0 人。

如果构造一个链：A-B-C，A 有 1 个朋友，B 有 2 个，C 有 1 个。
- 第 0 轮：没人离开
- 第 1 轮：A 和 C 离开（都有 1 个朋友）
- 第 2 轮：B 原来有 2 个朋友，但 A 和 C 都走了，B 现在有 0 个朋友。但第 0 轮已经过去了，B 不会在第 2 轮离开（第 2 轮是"有 2 个朋友的人"离开，B 现在只有 0 个朋友）。

所以 B 留下来了！答案是 1，符合样例。

## 寻找规律

从上面的分析，B 能留下来的关键是：当第 2 轮来临时，B 的朋友都已经走了，所以 B 不会被淘汰。

这提示我们一种构造方式：让某些人"保护"其他人。

考虑构造一个完全图（团）加上一些悬挂点：
- 一个大小为 $k$ 的团，团内每个人有 $k-1$ 个朋友
- 团外有 $n-k$ 个人，每个人都只和团内的某一个人连接（有 1 个朋友）

这样：
- 第 0 轮：团外的人不会离开（他们有 1 个朋友）
- 第 1 轮：团外的人离开（都有 1 个朋友）
- 第 2 轮及以后：团内的人原本有 $k-1$ 个朋友，但随着团外的人离开，他们的朋友数减少

等等，团内的人的朋友都在团内，团外的人离开不影响团内人的朋友数。所以团内的人始终有 $k-1$ 个朋友。

第 $k$ 轮：团内的人有 $k-1$ 个朋友，他们会离开！

所以最后团内的人也全走了，剩下 0 人。这不是最优的。

## 重新思考

让团内的人朋友数动态减少。考虑一条链或者一个星形结构？

换个思路：假设最后剩下 $k$ 个人，这 $k$ 个人必须满足什么条件？

他们中的每个人，在每一轮检查时，朋友数都不等于当前轮次。

如果最后剩下的人互相之间都没有朋友关系，那他们每人有 0 个朋友，会在第 0 轮离开，矛盾。

所以他们之间必须有朋友关系。假设最后剩下 $k$ 个人形成一个团，每人有 $k-1$ 个朋友。
- 第 0 轮：安全（都有朋友）
- ...
- 第 $k-1$ 轮：安全（都有 $k-1$ 个朋友，但还没轮到第 $k$ 轮）
- 第 $k$ 轮：危险！他们都有 $k-1$ 个朋友，而第 $k$ 轮检查的是"有 $k$ 个朋友的人"，安全
- 第 $k-1$ 轮：危险！他们都有 $k-1$ 个朋友，正好匹配

所以第 $k-1$ 轮他们会离开。要让他们留下来，需要第 $k-1$ 轮时他们的朋友数不等于 $k-1$。

这意味着他们中的一些朋友必须先离开，减少他们的朋友数。

## 正确构造

考虑这样构造：一个大小为 $k$ 的团，再加上一个额外的点连接团内的所有点。

总共 $k+1$ 个人。额外点有 $k$ 个朋友，团内每人有 $k$ 个朋友（团内 $k-1$ 个 + 额外点 1 个）。

第 0 轮到第 $k-1$ 轮：都没人离开（朋友数都是 $k$，大于轮次）
第 $k$ 轮：额外点有 $k$ 个朋友，离开！团内每人失去 1 个朋友，现在有 $k-1$ 个朋友。

但第 $k$ 轮已经过去了，第 $k-1$ 轮也过去了。团内的人不会再被检查 $k-1$ 个朋友的情况。

所以他们留下来了！用 $k+1$ 个人，可以保住 $k$ 个人。

反过来，用 $n$ 个人，可以保住 $n-1$ 个人？

验证 $n=3$：构造一个 2 人团 + 1 个额外点连接这两人。额外点有 2 个朋友，第 2 轮离开。团内两人原本各有 2 个朋友，额外点离开后各剩 1 个。但第 1 轮已经过去了，他们安全。剩下 2 人？

但样例答案是 1。哪里错了？

重新看：$n=3$，构造团大小为 2，额外 1 人。团内两人互相连接，额外点连接团内两人。
- A-B 相连，C-A 相连，C-B 相连
- A 的朋友：B, C（2 个）
- B 的朋友：A, C（2 个）
- C 的朋友：A, B（2 个）

三人都有 2 个朋友！第 2 轮三人一起离开。剩下 0 人。

问题：额外点连接团内所有人后，团内人的朋友数增加了。

修正构造：额外点只连接团内的一个人。
- A-B 相连，C-A 相连
- A 的朋友：B, C（2 个）
- B 的朋友：A（1 个）
- C 的朋友：A（1 个）

第 0 轮：安全
第 1 轮：B 和 C 离开（都有 1 个朋友）
第 2 轮：A 原来有 2 个朋友，现在 B 和 C 都走了，A 有 0 个朋友。但第 0 轮过去了，A 不会离开。

剩下 1 人。这就是样例答案！

## 推广

用 $n$ 个人，构造一个大小为 $k$ 的团，加上 $n-k$ 个悬挂点，每个悬挂点只连接团内的某一个特定点。

为了让团内的人能留下来，需要他们的朋友数在某一轮之前降到不等于轮次。

最优策略：让团大小为 2（一条边），其余 $n-2$ 个点都连接这条边的一个端点。
- 端点 A：连接 B 和 $n-2$ 个悬挂点，共 $n-1$ 个朋友
- 端点 B：只连接 A，共 1 个朋友
- 悬挂点：各连接 A，共 1 个朋友

第 1 轮：B 和所有悬挂点离开（共 $n-1$ 人），A 的朋友数变成 0
第 0 轮已过，A 安全留下。

这样剩下 1 人。但这不是最优的。

让团大小为 $k$，每个团内点连接一些悬挂点，使得团内各点的朋友数不同？

实际上，考虑让最后剩下的人形成一个团，他们在关键时刻朋友数刚好不等于检查值。

经过分析（或者打表找规律），答案是：
- $n = 1$：0 人（第 0 轮就离开）
- $n = 2$：0 人（两人互为朋友，第 1 轮一起离开）
- $n \ge 3$：$n - 2$ 人

构造：3 个人形成一个三角形（每人 2 个朋友），其余 $n-3$ 个人各连接三角形的一个顶点。

或者更简单的理解：最后最多剩下 $n-2$ 人，因为至少需要 2 个人作为"牺牲品"来让其他人留下来。

## 代码

```cpp
// 核心逻辑：n >= 3 时答案为 n - 2，否则为 0
int solve(int n) {
    if (n <= 2) return 0;
    return n - 2;
}
```

主函数读入 $t$ 组数据，每组输出 `max(0, n - 2)` 即可。

## 总结

核心要点：
1. 这是一道构造题，关键是设计朋友关系让尽可能多的人留下
2. 至少需要 2 个人作为牺牲品（他们会在某一轮离开）
3. 当 $n \ge 3$ 时，总能构造出剩下 $n-2$ 人的方案
4. 答案就是 $\max(0, n-2)$
