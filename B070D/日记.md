# 解题日记

今天分析一道关于树和路径的题。

题目是这样的：给定一棵有N个顶点的树，有N-1条边，每条边有距离。然后给定Q个查询和一个整数K，每个查询要求计算从顶点x经过顶点K到达顶点y的最短路径距离。

什么是树？题目说树是没有闭环的连通图，有N个节点和N-1条边。我想了想，这意味着任意两点间只有唯一路径。为什么？因为树是连通的，所以任意两点间至少有一条路径。又因为无环，所以不可能有两条不同的路径——如果有两条路径，它们就会形成一个环。这个性质很重要，它让我知道树上两点间的距离是确定的。

但"经过K"是什么意思？我直接看样例1来理解。

样例1输入：

```
5
1 2 1
1 3 1
2 4 1
3 5 1
3 1
2 4
2 3
4 5
```

第一行5表示5个节点。接下来4行是边，画出来应该是这样：

```
        1
       / \
      2   3
     /     \
    4       5
```

边权都是1。然后3和1表示3个查询，K等于1。再下面是三个查询：2 4、2 3、4 5。

输出是3、2、4。

现在分析第一个查询：从2到4，必须经过1。

如果直接走，2到4有边，距离是1。但要求经过1，那我得从2先到1，再从1到4。从1到4怎么走？1连2，2连4，所以是1→2→4。总路径是2→1→2→4，距离1+1+1=3。对上了，第一个输出是3。

第二个查询，从2到3，经过1。2→1→3，距离1+1=2。输出确实是2。

第三个查询，从4到5，经过1。4→2→1→3→5，距离1+1+1+1=4。输出是4。

所以我的理解是对的：必须经过K意味着路径被强制分成两段，从x到K，再从K到y。

规律出来了：答案 = dist(x,K) + dist(K,y)

现在想算法。如果对每个查询都现场算距离，比如从x沿着树的路径走到K，需要一步步找父节点或子节点，最坏情况下x和K分别在树的两端，要走O(N)步。Q有10^5个查询，总复杂度就是O(NQ)，也就是10^5 × 10^5 = 10^10，这太大了，肯定会超时。

注意到K是固定的。我可以从K出发，一次性算出K到所有节点的距离，存在数组里。这样每个查询直接查表相加，O(1)搞定。

用样例2验证这个方法：

```
7
1 2 1
1 3 3
1 4 5
1 5 7
1 6 9
1 7 11
3 2
1 3
4 5
6 7
```

7个节点，K=2。从2出发DFS，画个图：

```
    3(3) 4(5) 5(7) 6(9) 7(11)
      \   |   /   /   /
       \  |  /   /   /
        \ | /   /   /
          1
          |
         2(1)
```

1是中心，连2、3、4、5、6、7。从2开始：
- dist[2] = 0（起点）
- 走到1，dist[1] = 1
- 从1到3，dist[3] = 1+3 = 4
- 从1到4，dist[4] = 1+5 = 6
- 从1到5，dist[5] = 1+7 = 8
- 从1到6，dist[6] = 1+9 = 10
- 从1到7，dist[7] = 1+11 = 12

查询1：1→3，答案=dist[1]+dist[3]=1+4=5。输出是5，对。

查询2：4→5，答案=dist[4]+dist[5]=6+8=14。输出是14，对。

查询3：6→7，答案=dist[6]+dist[7]=10+12=22。输出是22，对。

再看样例3：

```
10
1 2 1000000000
2 3 1000000000
...
9 10 1000000000
1 1
9 10
```

10个节点排成一条链，边权都是10^9。K=1，查询9→10。

dist[9]=8×10^9，dist[10]=9×10^9，答案=17×10^9=17000000000。

这里要注意，距离超过了32位整数范围，需要用long long。

关于DFS的理解：从节点K开始遍历，相当于把K"拎"起来当根节点，整棵树就倒过来了。比如原来的树：

```
        1
       / \
      2   3
     /     \
    4       5
```

如果K=2，从2开始遍历，相当于把2拎起来：

```
    2(K)
   / \
  1   4
  |
  3
  |
  5
```

这时候2是根，1和4是2的子节点。父节点fa就是"从哪个节点走过来的"，用来避免走回头路。比如从2走到1，1的父节点就是2，遍历1的其他邻居时，遇到2要跳过。

最后整理解题步骤：

1. 建图
2. 从K开始DFS或BFS，记录dist数组
3. 对每个查询，输出dist[x]+dist[y]

代码：

```cpp
// u: 当前节点
// fa: 父节点（用来避免走回头路）
// d: 从起点K到当前节点u的距离
void dfs(int u, int fa, long long d) {
    dist[u] = d;  // 记录K到u的距离
    for (auto [v, w] : adj[u]) {  // 遍历u的所有邻接点，v是邻居，w是边权
        if (v != fa) {  // 不往回走
            dfs(v, u, d + w);  // 继续DFS，距离加上这条边的权重
        }
    }
}

// 从K开始，父节点设为0（不存在），初始距离0
dfs(K, 0, 0);

while (Q--) {
    cin >> x >> y;
    // x到y经过K = x到K + K到y
    cout << dist[x] + dist[y] << endl;
}
```

今天学到：遇到树的题目，先画个图帮助理解结构。样例是理解题意的关键，通过样例1我才明白"经过K"意味着路径被强制分段。算法设计时要分析复杂度，O(NQ)会超时就要想办法预处理。从固定起点算距离，一次DFS就能解决所有查询。另外注意数据范围，距离累加可能超过int范围，要用long long。
