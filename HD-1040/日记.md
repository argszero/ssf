# HD-1040 括号构造大赛

今天分析一道括号构造题。

## 题目描述

给定两个只包含左括号 `(` 和右括号 `)` 的字符串 S 和 T（长度都不超过 200）。需要构造一个新的括号串 f，满足三个条件：

1. f 是"合法括号串"
2. S 是 f 的子序列（不需要连续）
3. T 是 f 的子序列（不需要连续）

要求 f 的长度最短，输出这个最短长度。

**合法括号串的定义：**
- `()` 是合法括号串
- 如果 A 是合法括号串，那么 `(A)` 也是合法括号串
- 如果 A 和 B 都是合法括号串，那么 AB（拼接）也是合法括号串

简单说就是括号必须正确匹配，且任意前缀中左括号数量不少于右括号数量。

**子序列的定义：**
字符串 B 是 A 的子序列，意味着可以从 A 中按顺序选出一些字符（不要求连续），使得这些字符组成 B。

例如 `(()())` 包含 `()` 作为子序列（可以取第 0 和第 1 个字符 `(` 和 `)`）。

## 样例分析

**样例2：S=")", T="(("**

S 只有一个右括号，T 有两个左括号。要包含 S，f 中必须有 `)`；要包含 T，f 中必须有两个 `(`。

最短合法串是 `(())` 吗？验证：`(())` 长度为4，包含子序列 `()`（位置0,1或2,3），但不包含单独的 `)` 作为子序列（需要先有 `)` 前面没有匹配的 `(`）。

等等，`)` 作为子序列，只需要 f 中有 `)` 这个字符即可。`(())` 包含 `)`，是的。

但 `(())` 包含 `((` 吗？位置0,1是 `((`，是的。

所以 `(())` 长度为4，符合样例输出。

**样例3：S=")", T=")))"**

需要三个 `)`。最短合法串是 `((()))` 吗？长度为6。验证：包含 `)` 是的，包含 `)))` 是的（位置3,4,5）。符合样例。

## 问题转化

对于每个括号串，我们可以分析它需要的最少左括号和右括号数量。

但注意：S 和 T 本身可能不是合法括号串！比如单独的 `)` 就不是合法的。

这意味着 f 必须在 S 和 T 的字符之外，添加一些括号来使其合法。

## 关键观察

考虑一个括号串需要的"支撑"。对于任意括号串，我们可以计算：
- 需要在其前面加多少个 `(` 才能让它合法
- 需要在其后面加多少个 `)` 才能让它合法

对于一个串，设其任意前缀中右括号减左括号的最大值为 need_left（表示需要前面补这么多左括号），整个串的左括号减右括号的值为 balance（表示后面需要补这么多右括号）。

例如 `)`：前缀 `)` 的右减左为1，所以 need_left=1；整个串 balance = -1，所以 need_right=1。

例如 `((`：所有前缀右减左都<=0，need_left=0；balance=2，need_right=2。

## 构造 f 的方式

f 需要同时包含 S 和 T。有两种基本策略：

1. **串联**：S 和 T 依次排列，中间可能共享一些括号
2. **嵌套**：一个串在另一个串的内部

但更简单的方法是：考虑 f 的形式为：`(`...S...`)` 或者 `(`...T...`)`，或者是两者的某种组合。

实际上，可以枚举 S 和 T 在 f 中的相对位置。由于 f 是合法括号串，它可以写成 `(A)` 或 `AB` 的形式。

## DP 思路

设 dp[i][j] 表示包含 S[i..] 和 T[j..] 的最短合法括号串长度。

但这样状态可能不够，因为还需要知道当前已经有多少个未匹配的左括号（即当前处于什么"高度"）。

设 dp[i][j][k] 表示已经匹配了 S[0..i-1] 和 T[0..j-1]，当前有 k 个未匹配的左括号时，已经构造的 f 的最短长度。

转移：在 f 中添加一个 `(` 或 `)`。
- 添加 `(`：可以匹配 S[i]（如果 S[i]=='('）或 T[j]（如果 T[j]=='('），也可以都不匹配。高度 k+1。
- 添加 `)`：需要 k>0。可以匹配 S[i]（如果 S[i]==')'）或 T[j]（如果 T[j]==')'），也可以都不匹配。高度 k-1。

目标：dp[|S|][|T|][0]，即两个串都匹配完，且没有未匹配的左括号。

初始：dp[0][0][0] = 0。

复杂度：O(|S| * |T| * max_height)。max_height 最多是 |S|+|T|，所以是 200*200*400 = 16e6，可以接受。

## 代码核心

```cpp
// dp[i][j][k]：匹配了S前i个、T前j个，当前有k个未匹配左括号的最短长度
// 使用大数初始化
memset(dp, 0x3f, sizeof(dp));
dp[0][0][0] = 0;

for (int i = 0; i <= n; i++)
  for (int j = 0; j <= m; j++)
    for (int k = 0; k <= n+m; k++) {
      if (dp[i][j][k] == INF) continue;

      // 添加 '('
      int ni = i + (i < n && S[i] == '(');
      int nj = j + (j < m && T[j] == '(');
      dp[ni][nj][k+1] = min(dp[ni][nj][k+1], dp[i][j][k] + 1);

      // 添加 ')'
      if (k > 0) {
        ni = i + (i < n && S[i] == ')');
        nj = j + (j < m && T[j] == ')');
        dp[ni][nj][k-1] = min(dp[ni][nj][k-1], dp[i][j][k] + 1);
      }
    }

// 答案
return dp[n][m][0];
```

注意：这里有一个技巧——添加 `(` 时，如果 S[i] 或 T[j] 正好也是 `(`，就可以同时匹配掉。但两者不能同时匹配（一个字符只能匹配一次），所以需要仔细处理。

实际上上面的代码有问题：如果 S[i]=='(' 且 T[j]=='('，我们只能匹配其中一个，不能同时匹配两个。

正确的做法：枚举添加 `(` 后匹配 S[i] 还是 T[j] 还是都不匹配。但这样状态转移会复杂。

简化：我们可以不急着匹配，让 DP 自动选择最优。或者分别处理四种情况（匹配S、匹配T、都匹配（不可能）、都不匹配）。

实际上，如果 S[i]=='(' 且 T[j]=='('，我们只能选择匹配一个。所以转移要分情况：
- 匹配 S[i]（如果 S[i]=='('）
- 匹配 T[j]（如果 T[j]=='('）
- 都不匹配

这需要仔细实现。

## 总结

核心要点：
1. 三维 DP：dp[i][j][k] 表示匹配进度和当前括号深度
2. 转移时考虑添加 `(` 或 `)`，并检查是否能匹配 S 或 T 的当前字符
3. 注意 S[i] 和 T[j] 相同时只能匹配其中一个
4. 答案是 dp[|S|][|T|][0]

时间复杂度 O(|S| * |T| * (|S|+|T|))，空间复杂度相同，可以优化滚动数组但没必要。
